Reference: Sorting Methods

The following table defines the logic, required inputs, and specific behaviors for each sorting method available in the My Favorite Albums interface.

Method Name Description User Inputs / Filters Behavioral Notes
Number One Albums Displays the single highest-rated album for each year within a specified range. Year Range (Start Year to End Year) If a year in the range contains no data, that year is automatically skipped in the display.
Top Albums By Year Shows a complete ranked list of every album released in a specific year. Single Year Selection Manual Action: Users must manually resolve any ranking ties for the selected year.
Artists’ Albums Aggregates all favorited albums by a specific performer and provides a statistical summary. Artist Name Only artists already present in your imported dataset will appear as options.
Favorite Artists Calculates and ranks your top 15 highest-rated artists based on average album scores. Minimum Album Count; Toggle EPs/Live Albums Useful for filtering out "one-hit wonders" or non-studio recordings.
Artist Comparison Generates a line graph plotting the ratings of two specific artists over time. Two Artist Names Visualizes how your preference for an artist's discography has evolved compared to another.
Vinyl Identifies "missing" physical media by listing highly-rated albums you do not own. Minimum Rating Threshold (Min. 7) Only displays albums where the "Vinyl" column in your CSV is left blank.

Reference: CSV Columns

Column Name Description Requirement / Example
Year The year the album was released. Written as a number (e.g., 1996)
Ranking Your rank for that album within that specific year. 1 is your top favorite.
Album The official title of the album. e.g., Becoming X
Artist The name of the performer or group. e.g., Sneaker Pimps
Rating Your personal score for the music. Scale of 1 to 10
Vinyl Indicates physical vinyl ownership. Enter v if you do, or leave blank otherwise.
EP Indicates if the release is an EP. Enter EP if it is, or leave blank otherwise.
Live Indicates if it is a live recording. Enter Live if it is, or leave blank otherwise.

Reference: R & dplyr Data Functions

Use these functions within your custom .R scripts to process and filter your album data.

Function / Operator Purpose Syntax Example
filter() Returns rows that meet specific criteria. filter(album_data, Rating == 10)
select() Keep only specific columns; removes the rest. select(album_data, Artist, Album)
arrange() or arrange(desc()) Sorts data. Use desc() for descending order. arrange(desc(Year))
<- Assignment operator (stores data in a variable). my_variable <- 1993
== , > , < Comparison operators: Equals, Greater than, Less than. Year >= 2000

Reference: Troubleshooting & Common Errors

Use this table to diagnose issues when custom sorting methods or data files are not behaving as expected.

Issue Likely Cause Resolution
App fails to launch Script not "sourced." Ensure source("your_filename.R") is included at the top of app.R.
'Object not found' Case-sensitivity or typo. R is case-sensitive; check that album_data is not written as Album_Data.
Empty Table or Plot Criteria too strict. Check your CSV; if you filter for Rating == 11, no results will be returned.
Unexpected ',' error Syntax formatting error. Check your UI list; every element in a tabPanel needs a comma except the last one.
Graph doesn't update ID mismatch. Ensure the ID in plotOutput("my_id") matches output$my_id exactly.
