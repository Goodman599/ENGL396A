Creating and Adding a Sorting Method

If you want to get more technical and customize how you view your music, you need to create your own sorting method. This guide is for all skill levels, so don't worry if this is your first time coding!
By using dplyr, you can create new ways to filter and rank your data.

Creating your Analysis Method

Most analysis methods in My Favorite Albums involve taking your main dataset and "piping" it through a series of functions. This section will be a rundown on some important functions you may use during the filtering process.
We will imagine the .csv file as a spreadsheet, with each row as an album and each column as a data entry.

Warning:
If you don't know what a .csv file is, see here.

Creating your File

To start creating your own sorting method, first you need to create a file to house it in.

  1. In the bottom left of RStudio, make sure you are in MyFavoriteAlbums-main and click New File.
  2. Select R Script
  3. Name the file according to what you plan for the sorting method to do. For example, you can call a sorting method that sorts albums alphabetically as alphabetical_sort.R. Then click OK
Warning:
While programmers with no experience are included for this guide, it will not go over every basic programming principle. It will cover core concepts needed to work with My Favorite Albums, but following a general programming guide like this one by Bill Petti is highly encouraged.

Creating your Sorting Method(Function)

You need to give your sorting method, called functions, a name for My Favorite Albums to recognise it. The crux of a function is to process data, and you can have many different functions, usually called helper functions, in one file.

  1. In the editor on the top right of RStudio, enter the name of the function.
    Note:
    There are a few restrictions on what you can name your functions:
    • A function name must start with a letter and can be a combination of letters, digits, period(.), and underscore(_).
    • A function name cannot start with a number, period(.) or underscore (_)
    • Function names are case-sensitive (sorter, Sorter, and SORTER are three different functions)
  2. Type the assignment operator <-
  3. Enter the code function(){}
    • Between the parentheses(), you can enter variables separated by commas. Using it, you can pass data from elsewhere, such as selectors and sliders, to your functions.
    • Between the curly braces{}, you are free to add new lines, and is where you will be writing more code

Filtering the .csv Table

    • Warning:
    For the next functions, ensure your column names in the code match the headers in your album-rankings.csv exactly (e.g., "Rating" with a capital R).
  • Filtering by column(data type): The filter() function returns all of the rows that match a certain criteria.
    In the parenthesis, you must enter any data table. The data table storing all of your album rankings is stored in album_data. Follow that with each condition separated with commas.
    The filter() function creates a new data table with all of the valid entries, which can be fed into other functions.
    In R, there are 3 sets of operators you need to know:
    • ==: Equals to
    • >: Greater than and >=: greater than or equal to
    • <: Less than and <=: less than or equal to
    For example, to find all the albums you've rated 10/10, you can do:
    tenStarAlbums <- filter(album_data, Rating == 10) If you only want to find those released before 2000, you can do:
    tenStarAlbums <- filter(album_data, Rating == 10, Year < 2000)
    Note:
    Make sure your condition names are the exact same as the .csv column headers.
  • Displaying specific data: The select() function "trims" a data table down, removing unspecified columns.
    In the parenthesis, you must enter any data table. The data table storing all of your album rankings is stored in album_data. Follow that with each data entry you want to list separated with commas, from left to right.
    The select() function creates a new data table with all of the valid entries, which can be fed into other functions.
    If you just wanted to list every album with every artist, you can write:
    allAlbumsAndArtists <- select(album_data, Artist, Album)

To have your function return the data you calculated, assign a variable with that data as the last line of code in the function.

Incorporating New Methods into the App

Once you have written your custom R functions, you need to "wire" them into the main application. Because My Favorite Albums uses a modular structure, you will need to update three specific files: app.R, app_ui.R, and app_server.R.

Registering the Script (app.R)

The app.R file acts as the project manager. It needs to know your new script exists so it can load your functions into memory.

  1. Open app.R.
  2. Locate the list of source() commands at the top of the file.
  3. Add a new line for your file. For example:
    source("alphabetical_sort.R")
Note:
This step is crucial! If you don't source the file here, the app won't be able to find your function, even if the file is in the right folder.

Designing the Interface (app_ui.R)

The app_ui.R file controls the layout. You need to create a new "Tab" so users can actually see your analysis.

  1. Open app_ui.R and find the tabsetPanel section.
  2. Add a new tabPanel entry inside the list. It should look like this:
    tabPanel("Alphabetical Sort", htmlOutput("alpha_title"), tableOutput("alpha_table"))
Definition:
The tableOutput("alpha_table") is a placeholder. It tells the UI exactly where to draw the table once the server generates it.

Connecting the Logic (app_server.R)

The app_server.R file is the bridge between your data and the screen.

  1. Open app_server.R.
  2. Inside the server function, add a block to "render" your table:
    output$alpha_table <- renderTable({ return(your_function_name(album_data)) })
Warning:
The ID you use in the UI (e.g., "alpha_table") must match the ID in the Server (e.g., output$alpha_table) exactly, including capitalization!

Once these three steps are complete, click Run App in RStudio. Your new tab should appear with your custom analysis visible.
